# AIC-Turku Database: Agent Context (Current Snapshot)

This document is for automated AI agents (like Cursor, Copilot, or Claude) working in this repository. It intentionally describes only what exists **today** in this checkout to prevent hallucinations.

## 1) Project Purpose

This repository is the strict GitOps source of truth for AIC-Turku microscopy operations. 
State is stored as YAML ledgers and rendered dynamically into a static MkDocs dashboard and LLM-consumable JSON inventories.

Primary record classes:
- Instrument registry entries (`instruments/*.yaml`)
- QC session ledgers (`qc/sessions/**/<year>/*.yaml`)
- Maintenance event ledgers (`maintenance/events/**/<year>/*.yaml`)
- Controlled Vocabularies (`vocab/*.yaml`)

## 2) Current Repository Structure (Authoritative)

- `README.md` — Human-facing workflow + conventions overview.
- `instruments/` — Active instrument records.
  - `instruments/retired/` — Retired instrument records.
- `qc/sessions/` — QC session YAML files grouped by instrument + year.
- `maintenance/events/` — Maintenance event YAML files grouped by instrument + year.
- `vocab/` — **Strict controlled vocabularies.** This is the single source of truth for allowed terms (e.g., modalities, detectors). 
- `templates/` — Source templates for new records.
- `scripts/` — Automated pipelines:
  - `scripts/dashboard_builder.py` — The singular dashboard generation logic. Builds the `dashboard_docs/` output.
  - `scripts/validate.py` — Ledger validation and schema enforcement helpers.
  - `scripts/templates/` — Jinja templates used by the dashboard builder.
- `dashboard_docs/` — (Generated) The output directory for the MkDocs site, built dynamically by `dashboard_builder.py`.
- `mkdocs.yml` — Configuration for MkDocs (note: the `nav` is auto-generated by the build script).

*Note: Scripts like `build_docs.py` and static files like `docs/llm_microscopy_assistant.md` or `docs/vocabulary_dictionary.md` were deprecated and removed. Do not attempt to read or modify them.*
